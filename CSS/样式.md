> <h1 id="样式模块化">样式模块化</h1>

```less
.powerType {
  :global {
    .ant-radio-wrapper {
      line-height: 36px;
    }
  }
}
```

- **这段代码的含义是：**
	- 在一个局部样式（`.powerType`）作用域内，通过 `:global` 的方式对全局类名 `.ant-radio-wrapper` 进行样式覆盖，将其 `line-height` 设置为 `36px`。
	* `.ant-radio-wrapper` 是 Ant Design 的组件类名；
	* 所以你可以理解成：**为 `.powerType` 作用范围内的 `.ant-radio-wrapper` 设置 line-height 样式**。

<br/> 

**🟩 转换为普通 CSS 写法：**

CSS 中没有 `:global` 语法，因为它不做 CSS Modules 的作用域隔离。

你可以直接写成如下形式（等同效果）：

```css
.powerType .ant-radio-wrapper {
  line-height: 36px;
}
```

<br/> 

✅ 如何使用？


**📌 如果你使用的是 CSS Modules（如 `index.module.less`）：**

你的组件里应该这么写：

```tsx
import styles from './index.module.less';

return (
  <div className={styles.powerType}>
    <Radio.Group>
      <Radio value={1}>选项1</Radio>
    </Radio.Group>
  </div>
);
```

这样 `.powerType` 是局部类名，`.ant-radio-wrapper` 是全局类名，最终样式就会生效。

<br/><br/>

**📌 如果你不是用 CSS Modules，而是全局 CSS（如 `index.less`），可以直接写：**

```less
.powerType .ant-radio-wrapper {
  line-height: 36px;
}
```

然后组件中写法：

```tsx
import './index.less';

return (
  <div className="powerType">
    <Radio.Group>
      <Radio value={1}>选项1</Radio>
    </Radio.Group>
  </div>
);
```

> <h1 id="局部模块和全局模块化样式区别和影响">局部模块和全局模块化样式区别和影响</h1>


问题：为什么在 CSS Modules 的 `.module.less`（或 `.module.css`）文件中写

```css
:global {
  .powerType .ant-radio-wrapper {
    line-height: 36px;
  }
}
```

![css0.0.0.0.png](./../Pictures/css0.0.0.0.png)

**不能达到**你期望的效果，而类似这样写：

```less
.powerType {
  :global(.ant-radio-wrapper) {
    line-height: 36px;
  }
}
```

却是正确可用的？而对 `:global` 的使用不太理解。

<br/>

**解释核心**

- **`:global` 用法语法区别**

  在 CSS Modules 中，`:global` 既可以接“括号形式”写法，也可以接“块级形式”写法，但二者作用范围不同。

1. `:global(selector)`  
   — 只让括号中 `selector`（选择器）部分**全局生效**，但其**外层的选择器依然是局部(带hash)的**。

2. `:global { ... }`  
   — 里面的所有样式或选择器全部视为**全局**，不会加局部哈希。  
   > 这个写法其实是声明了一个“全局样式块”，里面所有选择器都不会被模块化处理。


<br/>

**我写的：**

```css
:global {
  .powerType .ant-radio-wrapper {
    line-height: 36px;
  }
}
```

**翻译效果是：**

- `.powerType .ant-radio-wrapper` 这个完整选择器都是全局的，不带模块哈希。
- 也就是说，CSS Modules 不会给 `.powerType` 加哈希，所以它只能匹配全局的 `.powerType` 类，而不是模块局部生成的类名。

但一般情况下，你的 `.powerType` 是通过 CSS Modules 管理的，它的实际类名被转译成了类似 `.powerType_hash` 的样式名，你想选中的是这个局部类，而不是全局的 `.powerType`。

所以，上面写法**失效的原因是它匹配全局 `.powerType`，而你React里真正用的是局部哈希名**。

<br/>

**推荐正确写法：**

```less
.powerType {
  :global(.ant-radio-wrapper) {
    line-height: 36px;
  }
}
```

**这段代码含义是：**

- `.powerType` 是模块化的，会带上 hash，比如 `.powerType_hash`
- 里面的 `.ant-radio-wrapper` 被 `:global()` 包裹，**不会带 hash**，是全局选择器
- 生成的 CSS 类选择器大致会是：
  ```css
  .powerType_hash .ant-radio-wrapper {
    line-height: 36px;
  }
  ```
- 这样就既能正确定位模块内的 `.powerType`，又能作用到第三方库或全局的 `.ant-radio-wrapper`，符合预期逻辑。

<br/>

**总结对比表**

| 写法                     | 作用范围及效果                                   | 典型用途                          |
|--------------------------|------------------------------------------------|----------------------------------|
| `:global { .a .b {}}`    | `.a` 和 `.b` 都是 *全局* 的，没有 hash          | 覆盖完全全局类、覆盖第三方全局样式  |
| `.a { :global(.b) {}}`  | `.a` 是局部（带hash），`.b` 是全局（没hash）   | 组件内局部类调用全局第三方类        |
| `.a :global .b {}`       | 也是局部 `.a` + 全局 `.b`                       | 同上                              |

<br/>

**参考示例（基于你场景）**

假设 React 组件代码：

```jsx
import styles from './style.module.less';

function MyComponent() {
  return (
    
      
        {/* 内容 */}
      
    
  );
}
```

你需要 `.powerType` 组件内部的 `.ant-radio-wrapper` 设置样式，建议：

```less
.powerType {
  :global(.ant-radio-wrapper) {
    line-height: 36px;
  }
}
```

这种写法会正确生成：

```css
.powerType_hash .ant-radio-wrapper {
  line-height: 36px;
}
```

这样 React 渲染时会生效。

<br/>

**额外说明**

- 直接用 `:global { .powerType .ant-radio-wrapper {}}` 会让 `.powerType` 也全局生效，如果你并没有全局定义 `.powerType`，样式根本匹配不到。
- CSS Modules 的核心目标是隔离局部样式，让父级保持哈希，`：global()` 用来声明**部分选择器全局有效**。
- 可以参考官方和社区对此的解释，[GitHub css-modules issue #264](https://github.com/css-modules/css-modules/issues/264)[1]。

<br/>

**简单回答**

**你的写法不生效，是因为 `:global {}` 让所有选择器包含的 `.powerType` 都变全局了，而真实 `.powerType` 是局部哈希名，不匹配全局。**

**正确做法是：**

```less
.powerType {
  :global(.ant-radio-wrapper) {
    line-height: 36px;
  }
}
```

让 `.powerType` 保持局部，里面的 `.ant-radio-wrapper` 才全局生效。这样才会有预期效果。

> <h1 id="样式文件处理流程">./css和.less文件处理流程</h1>

当在文件里写入 `import './index.less'` 到浏览器能看到样式，这几个工具（TypeScript编译检查、Webpack处理）分别在做什么。请看下面流程：

<br/> 

**⚡ 前端工程中 `.css` / `.less` 文件的处理流程**

```
┌──────────────────────────────┐
│  1. 你写代码                  │
│  import './index.less';      │
└───────────────┬──────────────┘
                │
                ▼
┌──────────────────────────────┐
│  2. TypeScript 检查           │
│  - 遇到 './index.less' 时，   │
│    先去看有没有声明模块       │
│  - typing.d.ts 里有：         │
│    declare module '*.less';   │
│  - 告诉 TS：这类文件合法      │
└───────────────┬──────────────┘
                │
                ▼
┌──────────────────────────────┐
│  3. 构建工具 (Webpack / Vite) │
│  - 根据配置找到 `less-loader` │
│    `css-loader` `style-loader`│
│  - 把 index.less 编译成 CSS   │
│  - 再把 CSS 转成 JS 模块       │
└───────────────┬──────────────┘
                │
                ▼
┌──────────────────────────────┐
│  4. 打包结果                  │
│  - JS 代码里会插入 <style> 标签│
│  - 或者把 CSS 单独抽出来 (插件)│
│  - 浏览器最终能解析样式        │
└──────────────────────────────┘
```

<br/>

 **🔑 关键点**

- **1.TypeScript 层**：
   只负责“不报错”，它不关心 CSS/LESS 里写了啥。
   → `declare module '*.css'` / `declare module '*.less'` 就是给 TS 一个“假的定义”。

- **2.Webpack/Vite 层**：
   真正干活的是构建工具。它们根据 loader/plugin 把 `.css` `.less` 转换成能运行的样子。


**3.运行时**：
   样式最终以 `<style>` 标签 或 单独的 `.css` 文件形式 注入到页面。

> <h1 id="overflow与阴影搭配">overflow与阴影搭配</h1>


- **1.`overflow` 属性的作用**

`overflow` 用来控制 **当元素内容超出容器盒子（即元素的 content box）时**，如何处理。

常见的取值有：

* **visible**（默认值） → 内容溢出会照常显示，不会被裁切。
* **hidden** → 超出的部分会被裁掉（不可见）。
* **scroll** → 不管内容是否超出，都出现滚动条。
* **auto** → 只有内容溢出时才出现滚动条。

<br/>

- **2.`overflow: hidden;` 的例子**

**裁剪溢出的内容**

```html
<div class="box">
  <p>这是一段很长的文字这是一段很长的文字这是一段很长的文字这是一段很长的文字这是一段很长的文字……</p>
</div>

<style>
.box {
  width: 200px;
  height: 50px;
  border: 1px solid #333;
  overflow: hidden; /* 超出200x50范围的文字会被裁掉 */
}
</style>
```

👉 效果：文字超过了 `200x50` 的盒子大小，超出的部分被 **隐藏**，不会显示出来。

<br/>

**配合 `box-shadow`**

```css
box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
overflow: hidden;
```

这里常见的用途是：

* **box-shadow** 给容器加上阴影，营造浮起的效果。
* **overflow: hidden** 把容器里的子元素超出范围的部分裁掉。

<br/>

比如在卡片组件中：

```html
<div class="card">
  <img src="https://via.placeholder.com/300x200" alt="image" />
  <p>卡片文字介绍</p>
</div>

<style>
.card {
  width: 200px;
  border-radius: 8px;                  /* 圆角 */
  box-shadow: 0 2px 8px rgba(0,0,0,0.1); /* 阴影 */
  overflow: hidden;                     /* 保证子元素（如图片）不会溢出圆角 */
}

.card img {
  width: 100%;
  display: block;
}
</style>
```

<br/>

**👉 效果：**

* 卡片有圆角和阴影。
* 如果没有 `overflow: hidden;`，图片的四个角可能超出卡片的圆角范围。
* 加上 `overflow: hidden;` 后，图片会被裁剪到圆角边界内，看起来更整齐。

<br/>

**3.详解 `overflow: hidden` 在实际开发中的常见用途**

1. **裁剪多余内容**：避免内容超出容器破坏布局。
2. **配合圆角使用**：裁剪子元素（特别是图片/背景），保持和父容器圆角一致。
3. **触发 BFC（Block Formatting Context）**：解决一些布局问题，比如父元素塌陷。

```css
.parent {
overflow: hidden; /* 可以让子元素浮动时父元素自动撑开 */
}
```

<br/>

**总结：**

* `overflow` → 控制内容溢出时的处理方式。
* `overflow: hidden` → 常用于裁剪内容 + 保证圆角效果 + 布局修复。
* 在 `box-shadow` 里一起用，主要是保证子元素不会突破圆角，让卡片更美观。

> <h1 id="媒体查询-最大宽度">媒体查询-最大宽度</h1>

```css
@media (max-width: 768px) {}
```

<br/>

**1.作用解释**

`@media (max-width: 768px)` 是 **媒体查询**，意思是：
👉 当 **屏幕宽度 ≤ 768px**（通常是平板或手机）时，里面的样式会生效。

* 你上面写的第一个：

  ```css
  @media (max-width: 768px) {
    .content-card {
      padding: 16px;
    }
  }
  ```

  这就表示：在小屏幕设备下，把 `.content-card` 的内边距改成 `16px`。

* 而第二个：

  ```css
  @media (max-width: 768px) {}
  ```

  里面是 **空的**，所以现在完全没有作用。

<br/>

**2.为什么可能会出现这种情况？**

一般是：

- 1.开发时先写了个 **空的 media 查询块**，准备后面加样式，忘了删。
- 2.有些代码生成器/模板会预留一个空的 media query，方便后续手写。

<br/>

- **3.结论**
	* **现在这段空的 `@media` 没有任何效果，可以删掉**。
	* 但如果你打算后面还要为小屏幕加新样式，可以保留作为占位。

> <h1 id="overflow-x-hidden-作用">overflow-x: hidden 作用</h1>

`overflow-x` 用来控制元素在**水平方向**内容溢出时的处理方式。

`overflow-x: hidden;` 的含义是：**水平超出容器宽度的内容会被裁剪隐藏，并且不会出现横向滚动条**。

对应垂直方向的属性是 `overflow-y`。

<br/>

**常见取值**

| 值 | 效果 |
|----|------|
| `visible` | 默认值，内容超出容器会露在外面，不裁剪、无滚动条 |
| `hidden` | 超出水平边界的内容直接裁剪隐藏，无横向滚动条 |
| `scroll` | 强制显示横向滚动条，不管内容是否超出 |
| `auto` | 内容超出时才出现横向滚动条 |

<br/>

**1.禁止页面横向滚动**

页面底部出现横向滚动条，通常是某个元素宽度超出视口。可以在 `body` 上隐藏横向溢出：

```css
body {
  overflow-x: hidden;
}
```

这会裁剪超出视口的部分，并消除横向滚动条。

<br/>

**2.轮播或滚动容器隐藏左右溢出内容**

```css
.wrap {
  width: 300px;
  overflow-x: hidden;
}

.list {
  width: 600px;
  display: flex;
}
```

`.list` 比父容器宽，超出 `300px` 可视区域的内容会被隐藏，常用于轮播图或横向滑动区域。

<br/>

**3.长文本不横向撑开盒子**

单行文字过长时，可以配合 `white-space` 和 `text-overflow` 做省略号：

```css
.box {
  width: 200px;
  white-space: nowrap;
  overflow-x: hidden;
  text-overflow: ellipsis;
}
```

<br/>

**注意点**

1. `overflow-x: hidden` 只会隐藏内容，不会压缩内容本身。
2. 父容器设置后，子元素绝对定位超出左右边界也会被裁剪。
3. `overflow: hidden` 等价于同时设置 `overflow-x: hidden` 和 `overflow-y: hidden`。
4. 移动端慎用。如果页面确实需要横向查看内容，隐藏溢出会导致内容不可见。

> <h1 id="媒体查询与系统深色模式">媒体查询与系统深色模式</h1>

`@media` 是 CSS **媒体查询（Media Query）**，作用是根据设备或系统条件，让一组样式在满足条件时才生效。

它可以理解为 CSS 里的条件判断，常见判断条件包括：屏幕宽度、屏幕高度、横竖屏、系统深浅色模式、打印设备等。

<br/>

**基础示例**

窗口宽度小于等于 `768px` 时，字体变小：

```css
@media (max-width: 768px) {
  body {
    font-size: 14px;
  }
}
```

<br/>

**`prefers-color-scheme: dark` 的含义**

```css
@media (prefers-color-scheme: dark) {
  .guide-container {
    background-color: #1a1a1a;
  }
}
```

`prefers-color-scheme` 是一个媒体特性，用来读取**操作系统或浏览器的主题设置**。

| 值 | 含义 |
|----|------|
| `light` | 系统处于浅色模式 |
| `dark` | 系统处于深色或夜间模式 |

整句含义是：**当用户电脑或手机系统设置为深色模式时，执行大括号内的样式**。

系统深色模式开启时，`.guide-container` 背景变成 `#1a1a1a`；系统浅色模式下，这段样式不生效。

<br/>

**完整示例**

```css
/* 默认浅色样式 */
.guide-container {
  background-color: #ffffff;
}

/* 系统深色模式覆盖 */
@media (prefers-color-scheme: dark) {
  .guide-container {
    background-color: #1a1a1a;
    color: #eee;
  }
}

/* 可选：单独匹配浅色模式 */
@media (prefers-color-scheme: light) {
  .guide-container {
    background-color: #fff;
  }
}
```

<br/>

**特点**

1. 可以跟随系统主题自动切换，不需要 JS。
2. Windows、macOS、iOS、Android 主流环境都支持。
3. 媒体查询内样式在满足条件时会覆盖普通样式。
4. 只能跟随系统主题；如果网页需要独立主题切换，通常还需要 JS 配合 class 或属性实现。

<br/>

**区分**

- `@media (max-width: 1200px)`：按屏幕尺寸修改样式，常用于响应式布局。
- `@media (prefers-color-scheme: dark)`：按系统深浅色主题修改样式，常用于暗黑模式。

> <h1 id="同名-css-选择器合并">同名 CSS 选择器合并</h1>

如果两段 CSS 使用同一个选择器，例如都写 `.video-list`，浏览器会把它们的属性**叠加合并**。只要属性不冲突，就会同时生效。

例如下面两段：

```css
.video-list {
  display: flex;
  flex-direction: column;
  gap: 20px;
}

.video-list {
  will-change: transform;
  contain: layout style paint;
}
```

最终效果等价于：

```css
.video-list {
  display: flex;
  flex-direction: column;
  gap: 20px;
  will-change: transform;
  contain: layout style paint;
}
```

<br/>

**第一段：控制布局**

```css
.video-list {
  display: flex;
  flex-direction: column;
  gap: 20px;
}
```

1. `display: flex`：开启弹性布局。
2. `flex-direction: column`：子元素垂直上下排列。
3. `gap: 20px`：列表项之间保持 `20px` 间距。

<br/>

**第二段：优化性能**

```css
.video-list {
  will-change: transform;
  contain: layout style paint;
}
```

`will-change: transform` 用来提前告诉浏览器：这个元素后续可能会做位移、滚动或动画变换。浏览器可能会提前做渲染优化，让滚动或动画更流畅。

`contain: layout style paint` 用来把元素隔离成相对独立的渲染区域：

| 值 | 作用 |
|----|------|
| `layout` | 内部布局变化尽量不影响外部布局 |
| `style` | 内部样式计算尽量不影响外部元素 |
| `paint` | 重绘尽量限制在这个元素内部 |

这种写法适合长列表、视频列表、无限滚动列表等容易触发大量渲染的场景。

<br/>

**总结**

- `flex`、`flex-direction`、`gap` 控制列表怎么排列。
- `will-change`、`contain` 优化列表滚动或动画性能。
- 两段同名选择器不会互相覆盖，除非它们设置了同一个属性；同一属性后写的会覆盖先写的。

`will-change` 不要全局滥用，只应加在确实会滚动、位移或动画变化的元素上，否则可能占用更多内存。

> <h1 id="背景模糊层与-android-webview-兼容">背景模糊层与 Android WebView 兼容</h1>

下面这类样式常用于视频播放页的背景模糊层：把底部的视频列表做成虚化背景，营造沉浸式播放效果，同时规避 Android WebView 的视频层兼容问题。

```css
.player-page-background {
  min-height: 100vh;
  filter: blur(18px);
  transform: scale(1.03);
}
```

<br/>

**核心属性**

1. `min-height: 100vh`：保证背景层最小高度等于一整屏高度，避免内容不足时底部露白。
2. `filter: blur(18px)`：给整个元素添加高斯模糊，让列表内容成为虚化背景，不抢视频主体视觉焦点。
3. `transform: scale(1.03)`：把元素放大到原尺寸的 `1.03` 倍，用来遮住 `blur` 后四周可能出现的半透明虚边。

<br/>

**设计意图**

背景层本质上是页面底部的视频列表，只是被放在最底层作为模糊背景使用，上层再放真正的视频播放器。

如果背景层里还挂载广告占位，原生广告可能在背景中留下空白块，破坏整体虚化效果，所以这类背景层通常不挂载广告占位。

<br/>

**Android WebView 兼容处理**

Android WebView 中存在一种常见问题：如果把 `filter: blur(...)` 加在覆盖于视频上方的遮罩层上，浏览器可能错误地把底下的 `video` 图层也一起模糊。

更稳妥的做法是：**只模糊底部背景列表，不模糊覆盖在视频上的遮罩层**。这样可以从层级上规避 WebView 把滤镜错误应用到视频图层的问题，保证视频画面清晰。

<br/>

**常见三层结构**

1. 最底层：`.player-page-background`，虚化的视频列表背景。
2. 中间层：真正的 `video` 播放器，保持清晰显示。
3. 最上层：播放按钮、进度条、返回按钮等交互控件。

> <h1 id="全屏固定播放层">全屏固定播放层</h1>

播放器悬浮承载层常见写法如下：

```css
.player-layer {
  position: fixed;
  inset: 0;
  z-index: 900;
  display: flex;
  align-items: center;
  justify-content: center;
  padding: 0;
}
```

整体效果是：让播放容器**固定悬浮在整个屏幕最上层、铺满全屏，并让内部播放器居中显示**。

<br/>

**1.`position: fixed` 固定定位**

`position: fixed` 会让元素脱离普通文档流，并相对于浏览器可视窗口进行定位。

核心特性：

1. 不随页面滚动移动，始终固定在视口同一位置。
2. 脱离文档流，不占用原来的页面位置。
3. 定位参照物是浏览器可视窗口，而不是父元素。

在播放页中，它可以保证用户滚动底部视频列表时，中间的播放器不会跟着滚动消失。

| 定位方式 | 参照物 | 滚动表现 |
|----------|--------|----------|
| `position: absolute` | 最近的非 `static` 定位父元素 | 父元素滚动时会跟着动 |
| `position: fixed` | 浏览器可视窗口 | 页面滚动时位置不变 |

<br/>

**2.`inset: 0` 四周偏移简写**

`inset` 是 CSS 简写属性，用来一次性设置 `top`、`right`、`bottom`、`left`。

```css
inset: 0;
```

等价于：

```css
top: 0;
right: 0;
bottom: 0;
left: 0;
```

它会让播放层的四条边都贴齐视口边缘，形成全屏覆盖层。

<br/>

**3.`z-index: 900` 层叠层级**

`z-index` 控制元素在 Z 轴上的堆叠顺序，数值越大越靠上。

它需要配合非默认定位生效，例如这里的 `position: fixed`。`z-index: 900` 可以让播放器覆盖普通页面内容、底部列表、导航栏等元素，但仍然给更高层级的全局弹窗或提示框留出空间。

常见层级约定：

| 场景 | 示例层级 |
|------|----------|
| 普通内容 | `1` |
| 顶部导航栏 | `100` |
| 悬浮播放层或侧边栏 | `900` |
| 全局弹窗、消息提示 | `1000` 到 `9999` |

<br/>

**居中相关属性**

- `display: flex`：开启弹性布局。
- `align-items: center`：垂直方向居中。
- `justify-content: center`：水平方向居中。
- `padding: 0`：去掉内边距，避免全屏层出现额外留白。

> <h1 id="退出全屏-svg-图标">退出全屏 SVG 图标</h1>

下面是一个视频播放器常见的**退出全屏图标**，视觉语义是“四角向内收起”。

<br/>

**CSS 样式**

```css
.exit-fullscreen-icon {
  display: block;
  flex-shrink: 0;
}
```

1. `display: block`：把 SVG 设置为块级元素，消除行内元素默认的底部留白，让宽高表现更稳定。
2. `flex-shrink: 0`：在 flex 控制栏中禁止图标被压缩，确保图标始终保持原始尺寸。

<br/>

**SVG 外层属性**

```jsx
<svg
  className="exit-fullscreen-icon"
  width="20"
  height="20"
  viewBox="0 0 20 20"
  aria-hidden="true"
  focusable="false"
>
```

| 属性 | 作用 |
|------|------|
| `className` | JSX 写法，对应 HTML 的 `class`，用于绑定 CSS 样式 |
| `width="20" height="20"` | 图标实际显示尺寸为 `20px * 20px` |
| `viewBox="0 0 20 20"` | 定义 SVG 内部坐标系，支持矢量无损缩放 |
| `aria-hidden="true"` | 告诉屏幕阅读器这个图标是装饰元素，不需要朗读 |
| `focusable="false"` | 禁止 SVG 本身获得键盘焦点，让焦点停留在外层按钮上 |

按钮语义通常应写在外层按钮上，例如：

```jsx
<button aria-label="退出全屏">
  {/* svg icon */}
</button>
```

<br/>

**`path` 路径与样式**

```jsx
<path
  d="M7 3v4H3M13 3v4h4M17 13h-4v4M3 13h4v4"
  fill="none"
  stroke="currentColor"
  strokeWidth="2"
  strokeLinecap="round"
  strokeLinejoin="round"
/>
```

| 属性 | 作用 |
|------|------|
| `fill="none"` | 不填充颜色，只显示描边线条 |
| `stroke="currentColor"` | 描边颜色继承父元素的 `color` |
| `strokeWidth="2"` | 线条粗细为 `2` |
| `strokeLinecap="round"` | 线条端点使用圆角 |
| `strokeLinejoin="round"` | 线条拐角使用圆角 |

<br/>

**路径含义**

| 路径片段 | 绘制逻辑 | 对应位置 |
|----------|----------|----------|
| `M7 3 v4 H3` | 从 `(7,3)` 向下画到 `(7,7)`，再向左画到 `(3,7)` | 左上角向内 |
| `M13 3 v4 h4` | 从 `(13,3)` 向下画到 `(13,7)`，再向右画到 `(17,7)` | 右上角向内 |
| `M17 13 h-4 v4` | 从 `(17,13)` 向左画到 `(13,13)`，再向下画到 `(13,17)` | 右下角向内 |
| `M3 13 h4 v4` | 从 `(3,13)` 向右画到 `(7,13)`，再向下画到 `(7,17)` | 左下角向内 |

四个角组合后，就是退出全屏常见的“四角向中心收起”图标。
